Cache Statistics

For some aggregations, information is stored in Mongo DB, which you can configure in the configuration panel. If your Mongo configuration is working properly, a new database called redi will be created. The aggregations results are in statistics collection. Each result of a query is stored as a document. Until now, there are five queries preprocessed.

  • barchar: contains information of total of authors and publications for each institution. This information is used in the barchar view within REDI.
  • count_authors: Total of authors on each institution.
  • count_publications: Total of publications on each institution.
  • count_research_areas: Frequency of research areas.
  • keywords_frequencypub_gt4: keywords than have a frequency greater than four.
